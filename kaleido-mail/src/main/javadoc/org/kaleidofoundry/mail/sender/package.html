Service d'envoi

Ce package permet une gestion simplifiée des envoies de mail. Le service principal est MailSenderService. Les mails envoyés sont de type MailMessage. Ce service possède plusieurs implémentations accessibles via la classe MailSenderFactory :

Implémentations disponibles :

  • 1. MailSenderServiceSessionImpl : Cette implémentation permet l’envoie synchrone de MailMessage. Elle encapsule un MailSessionService. A l’instanciation de la classe, un MailSessionContext est passé au constructeur afin que le service sache quel fournisseur de Session instancier et utiliser pour envoyer les mails. Cette implémentation permet un envoi direct, mais ne permet pas une encapsulation JTA de la transaction.
  • 2. MailSenderServiceEjbImpl : Cette implémentation permet l’envoie synchrone de MailMessage via un EJB (accédant à une Session définit sur le serveur d’application). L’avantage de cette solution est un envoie direct, avec possible utilisation de JTA. Cela permet donc une meilleur gestion des transactions en cas de problèmes d’envoie (transaction de plus haut niveau pour l’utilisateur).
    Pour être utilisé, l’EJB EjbMailSender doit être déployé et accessible sur un serveur d’application.
  • 3. MailSenderServiceJmsImpl : Cette implémentation permet l’envoie de MailMessage via le procole JMS (envoi asynchrone, dans une queue de traitement). De cette manière le processus d’envoi de mail peu être envoyé et confié à un service externe au process en cours (par exemple un serveur d’application tierce capable de traiter une plus grande charge d’envoie).
    Pour être utilisé, l’EJB EjbMailJmsReceiver doit être déployé et accessible sur un serveur d’application.
